## 0. 이 문서의 역할

코드베이스를 **4단계 줌**으로 설명한다.

| C4 Level | 질문 | 이 프로젝트에서 보면 |
| -------- | ---- | ------------------ |
| **1 Context** | 시스템이 누구와 무엇을 주고받나? | 운영자·고객 ↔ 할인 계산기 |
| **2 Container** | 실행 단위는 무엇인가? | Flask 앱, 도메인 모듈, pytest |
| **3 Component** | 컨테이너 안의 주요 부품은? | `subtotal`, `final_total`, 라우트 핸들러 |
| **4 Code** | 클래스·함수 수준은? | `cart.py` / `app.py` 공개 API |

> 이 저장소는 **ECB**를 C4 Container/Component에 매핑한다.  
> **Entity** = 도메인 컴포넌트 · **Boundary** = HTTP/UI 컴포넌트 · **Control** = 별도 모듈 없음(Flask 라우트가 얇은 조정 역할).

## 1. Level 1 — System Context

시스템 경계 밖의 액터와 시스템의 관계.

레이아웃: `UpdateLayoutConfig($c4ShapeInRow="3")` + 선언 순서(좌 Person → System → 우 Person → 아래 Person).

```mermaid
C4Context
    title System Context — 장바구니 할인 계산기

    UpdateLayoutConfig($c4ShapeInRow="3", $c4BoundaryInRow="1")

    Person(operator, "운영 담당자", "주문 금액·할인 검산")
    System(cart_system, "장바구니 할인 계산기", "소계·문턱 할인·VIP 할인 계산 및 입력 검증")
    Person(customer, "고객", "단가·수량·VIP 여부 입력")
    Person(developer, "개발자", "TDD·계약 ID 기반 구현")

    Rel(operator, cart_system, "검산·운영 확인", "브라우저 / pytest")
    Rel(customer, cart_system, "price, qty, VIP 제출", "HTTP 폼")
    Rel(developer, cart_system, "RED→GREEN→REFACTOR", "pytest, IDE")
```

### 1.1 시스템 책임

- 장바구니 품목(`price`, `qty`)으로 **소계** 계산
- 소계 **≥ 50,000원**이면 10% 문턱 할인 (`round`)
- **VIP** 고객이면 문턱 할인 **후** 5% 추가 할인
- 잘못된 입력(음수, None, 비숫자 qty) **명시적 거부**

### 1.2 시스템 밖에 두는 것 (OOS)

쿠폰, 배송비, 세금, 포인트, DB, 결제 게이트웨이 — [Discovery §6](./01.product-discovery-contracts.md) 참조.

## 2. Level 2 — Container

시스템을 구성하는 **배포·실행 단위**.

```mermaid
C4Container
    title Container — 장바구니 할인 계산기

    Person(customer, "고객")
    Person(developer, "개발자")

    Container_Boundary(app_boundary, "cursor-tdd-cart") {
        Container(web_app, "Flask Web App", "Python, Flask 3.1", "HTTP 폼·입력 파싱·응답")
        Container(domain, "Cart Domain", "Python 3.12", "순수 할인 계산·불변식")
        Container(test_runner, "pytest Suite", "pytest 9.1", "Entity/Boundary 계약 검증")
    }

    Rel(customer, web_app, "GET /, POST /calc", "HTTP")
    Rel(web_app, domain, "final_total(items, is_vip)", "Python import")
    Rel(developer, test_runner, "pytest -q", "CLI")
    Rel(test_runner, domain, "직접 호출", "import src.cart")
    Rel(test_runner, web_app, "test_client", "import src.app")
```

### 2.1 Container 요약

| Container | 경로 | 기술 | 책임 |
| --------- | ---- | ---- | ---- |
| **Flask Web App** | `src/app.py` | Flask | Boundary — HTTP, 폼 파싱, UE-1 |
| **Cart Domain** | `src/cart.py` | 순수 Python | Entity — INV-*, E-1, E-2 |
| **pytest Suite** | `tests/` | pytest | Dual-Track 계약 검증 |

### 2.2 의존 방향 (필수)

```
tests/entity  ──►  src/cart.py
tests/boundary ──►  src/app.py  ──►  src/cart.py

src/cart.py  ✗──►  Flask / app.py   (금지)
```

Entity는 프레임워크를 모른다. Boundary만 Entity를 import 한다.

## 3. Level 3 — Component

### 3.1 Cart Domain (`src/cart.py`)

```mermaid
C4Component
    title Component — Cart Domain (Entity)

    Container_Boundary(domain, "src/cart.py") {
        Component(validate, "_validate_line_items", "Python fn", "E-2: 음수 price/qty 검증")
        Component(subtotal_c, "subtotal", "Python fn", "INV-1: Σ(price×qty), E-1/E-2 호출")
        Component(threshold_c, "apply_threshold_discount", "Python fn", "INV-2: 문턱 10% 할인")
        Component(final_c, "final_total", "Python fn", "INV-3, INV-4: VIP·최종 금액")
        Component(constants, "THRESHOLD, THRESHOLD_RATE", "상수", "INV-2 SSOT")
    }

    Rel(subtotal_c, validate, "호출")
    Rel(final_c, subtotal_c, "소계 조회")
    Rel(final_c, threshold_c, "문턱 할인 적용")
    Rel(threshold_c, constants, "읽기")
```

| Component | 계약 ID | 입력 → 출력 |
| --------- | ------- | ----------- |
| `_validate_line_items` | E-2 | `items` → void / `ValueError` |
| `subtotal` | INV-1, E-1, E-2 | `items: list[dict]` → `int` |
| `apply_threshold_discount` | INV-2 | `amount: int` → `int` |
| `final_total` | INV-3, INV-4 | `items, is_vip` → `int` |

### 3.2 Flask Web App (`src/app.py`)

```mermaid
C4Component
    title Component — Flask Web App (Boundary)

    Container_Boundary(web, "src/app.py") {
        Component(index, "index()", "GET /", "UC-1: 주문 폼 HTML")
        Component(calc, "calc()", "POST /calc", "UC-2, UE-1: 파싱·계산·응답")
        Component(flask_app, "Flask app", "Framework", "라우팅·request 객체")
    }

    Container_Ext(domain, "Cart Domain", "final_total")

    Rel(index, flask_app, "등록")
    Rel(calc, flask_app, "등록")
    Rel(calc, domain, "final_total(items, is_vip)", "위임")
```

| Component | 계약 ID | 동작 |
| --------- | ------- | ---- |
| `index()` | UC-1 | `price`, `qty`, `vip` 폼 반환 |
| `calc()` | UC-2, UE-1 | 폼 → `items` 변환 → `final_total` → 문자열 응답 |

### 3.3 Test Suite (`tests/`)

```mermaid
C4Component
    title Component — Dual-Track Tests

    Container_Boundary(tests, "tests/") {
        Component(entity_tests, "tests/entity/test_cart.py", "Track B", "INV-1~4 직접 검증")
        Component(boundary_tests, "tests/boundary/test_app.py", "Track A", "UC-*, UE-1, E-*")
    }

    Container_Ext(cart, "cart.py", "Entity API")
    Container_Ext(app, "app.py", "Flask routes")

    Rel(entity_tests, cart, "import")
    Rel(boundary_tests, app, "test_client")
    Rel(boundary_tests, cart, "E-1, E-2 진입점")
```

| Track | 경로 | 검증 대상 |
| ----- | ---- | --------- |
| **B (Entity)** | `tests/entity/` | 도메인 불변식 INV-* |
| **A (Boundary)** | `tests/boundary/` | HTTP 계약 UC-*, UE-1 + Entity 에러 E-* |

## 4. Level 4 — Code

### 4.1 공개 API (Entity)

```python
# src/cart.py — 프레임워크 무의존

THRESHOLD: int = 50_000      # INV-2
THRESHOLD_RATE: float = 0.9  # INV-2

def subtotal(items: list[dict]) -> int: ...
def apply_threshold_discount(amount: int) -> int: ...
def final_total(items: list[dict], is_vip: bool = False) -> int: ...
```

**품목 스키마** (암묵적 계약):

```python
item = {"price": int, "qty": int}
items = list[item]
```

### 4.2 Boundary 라우트

```python
# src/app.py

GET  /       → index()   # UC-1
POST /calc   → calc()    # UC-2, UE-1
```

**폼 필드 SSOT**: `price`, `qty`, `vip`(checkbox), `action=/calc`

### 4.3 할인 파이프라인 (핵심 흐름)

```mermaid
flowchart LR
    A[items] --> B[subtotal]
    B --> C{amount >= 50_000?}
    C -->|Yes| D["round(amount × 0.9)"]
    C -->|No| E[amount 그대로]
    D --> F{is_vip?}
    E --> F
    F -->|Yes| G["round(× 0.95)"]
    F -->|No| H[결제 금액]
    G --> H
```

**예시** (UC-2 테스트 케이스):

```
price=60000, qty=1, vip=on
  → subtotal = 60000
  → threshold = round(60000 × 0.9) = 54000
  → vip       = round(54000 × 0.95) = 51300
```

### 4.4 에러 전파

| 위치 | 조건 | 결과 |
| ---- | ---- | ---- |
| Entity `subtotal` | `items is None` | `TypeError` (E-1) |
| Entity `_validate_line_items` | `price` or `qty` < 0 | `ValueError` + index (E-2) |
| Boundary `calc` | `qty` 비숫자 | HTTP 400 `"invalid qty"` (UE-1) |
| Boundary `calc` | Entity 예외 미처리 | 500 가능 *(음수 price 등 — 현재 미검증)* |

## 5. 계약 ID × C4 매핑

| 계약 ID | C4 Component | 테스트 |
| ------- | ------------ | ------ |
| INV-1 | `subtotal` | `test_inv_1_*` |
| INV-2 | `apply_threshold_discount` | `test_inv_2_*` |
| INV-3 | `final_total` (VIP 분기) | `test_inv_3_*` |
| INV-4 | `final_total` (전체) | `test_inv_4_*` |
| E-1 | `subtotal` | `test_e_1_*` (boundary) |
| E-2 | `_validate_line_items` | `test_e_2_*` (boundary) |
| UC-1 | `index()` | `test_uc_1_*` |
| UC-2 | `calc()` | `test_uc_2_*` |
| UE-1 | `calc()` qty 파싱 | `test_ue_1_*` |

계약 ID가 테스트·구현·이 문서를 잇는 **추적 키**다. ID 없는 동작은 구현하지 않는다.

## 8. 설계 결정 기록 (ADR 요약)

| 결정 | 이유 |
| ---- | ---- |
| ECB / Entity–Boundary 분리 | 할인 규칙을 HTTP와 독립 검증 (pytest entity track) |
| 계약 ID 주석 | C2C 추적 — RED/GREEN/REFACTOR 동일 ID 참조 |
| `round()` 사용 | 엑셀 검산 `(ROUND)` 근거 |
| 문턱 `>= 50_000` | README·테스트 경계값 50,000 → 45,000 |
| 단일 품목 폼 | UC-2 범위 — 다품목은 미계약 |
| E-1/E-2를 Entity에서 검증 | 실습 범위 — Boundary* 계약을 진입점에서 실행 |

## 9. 확장 시 C4 변경 가이드

| 추가 기능 | 영향 Container/Component | 선행 조건 |
| --------- | ------------------------ | --------- |
| 다품목 폼 | `calc()` 파싱 확장 | 신규 UC-* 계약 + Discovery |
| Discovery E-2 (빈 POST) | `calc()` 검증 강화 | RED 테스트 in boundary |
| 쿠폰 | `cart.py` 신규 component | OOS 해제 + AC/INV 정의 |

**규칙**: Component 추가 전 `Report/01.*` 또는 PRD에 계약 ID를 먼저 정의한다.
