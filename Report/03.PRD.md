## 1. 개요

### 1.1 제품명
**장바구니 할인 계산기** (Cart Discount Calculator)

### 1.2 한 줄 정의
온라인 주문 장바구니의 **소계·문턱 할인·VIP 할인**을 계산하고, 잘못된 입력을 거부하는 **도메인 로직 + 최소 웹 폼**을 제공한다.

### 1.3 배경
운영·CS·엑셀 검산에서 확인된 실제 사례를 바탕으로, 추측 없이 **테스트 가능한 계약(ID)** 만 구현한다. 개발 방식은 Dual-Track TDD(RED → GREEN → REFACTOR)이며, 계약 ID가 요구사항·테스트·구현을 연결한다.

### 1.4 현재 구현 상태

| 영역 | 상태 | 비고 |
| ---- | ---- | ---- |
| Entity (`cart.py`) | 구현 완료 | INV-1~4, E-1, E-2 |
| Boundary (`app.py`) | 부분 구현 | UC-1, UC-2, UE-1 |
| Discovery E-2 (빈 POST → 500 방지) | 미구현 | `/order` 라우트 없음 |

---

## 2. 문제 정의

- 주문 소계 **5만 원 이상** 시 10% 할인 적용
- **VIP** 고객은 문턱 할인 **이후** 추가 5% 할인
- 음수 수량 입력 시 0원 계산 금지 (CS 클레임)
- 웹 폼 제출 시 서버 500 오류 방지

---

## 3. 목표 및 비목표

### 3.1 목표
1. 검증된 사례와 일치하는 결제 금액 계산
2. 할인 순서 고정: 문턱 → VIP
3. 잘못된 입력 명시적 거부
4. 계약 ID 기반 추적 (INV-*, E-*, UC-*)
5. ECB 분리 (Entity는 Flask 비의존)

### 3.2 비목표 (OOS)

| ID | 제외 | 사유 |
| -- | ---- | ---- |
| OOS-2 | VIP 단독 할인 | 문턱 후만 근거 |
| OOS-3~5 | 쿠폰·배송비·세금·포인트 | 비범위 |
| OOS-6 | 빈 장바구니 Entity 동작 | 사례 미제공 |
| OOS-8~9 | 소수 단가·floor/ceil | 정수 원·ROUND만 확인 |

> OOS-1(50,000 정확히): 현재 구현은 `>= 50_000` 경계 포함.

---

## 4. 기능 요구사항

### 4.1 Entity (`src/cart.py`)

| ID | 규칙 |
| -- | ---- |
| INV-1 | `subtotal = Σ(price × qty)` |
| INV-2 | `amount >= 50_000` → `round(amount × 0.9)` / 미만 → 그대로 |
| INV-3 | VIP: 문턱 결과에 `round(× 0.95)`, 순서 문턱→VIP |
| INV-4 | `0 ≤ final_total ≤ subtotal` |
| E-1 | `items is None` → `TypeError` |
| E-2 | 음수 price/qty → `ValueError` (인덱스 포함) |

**검증 앵커**

| 소계 | VIP | 결제 |
| ---- | --- | ---- |
| 66,000 | No | 59,400 |
| 60,000 | No | 54,000 |
| 48,000 | No | 48,000 |
| 66,000 | Yes | 56,430 |

### 4.2 Boundary (`src/app.py`)

| ID | 규칙 |
| -- | ---- |
| UC-1 | `GET /` → 200, price·qty·vip 폼 |
| UC-2 | `POST /calc` → `final_total` 위임, 결과 표시 |
| UE-1 | 비숫자 qty → 400 |

---

## 5. 아키텍처

```
Boundary (app.py)  →  Entity (cart.py)
  GET /, POST /calc     subtotal, apply_threshold_discount, final_total
```

Entity는 Flask를 import하지 않는다. Boundary는 할인 공식을 재구현하지 않는다.
